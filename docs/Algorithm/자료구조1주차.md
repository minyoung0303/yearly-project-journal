## 1️⃣ Array vs List (배열 vs 리스트)

### ✅ Array (배열)

- **연속된 메모리 공간**
- 크기 **고정**
- 인덱스 접근 빠름 → **O(1)**

❌ 단점

- 중간 삽입/삭제 느림 → **O(N)**

📌 사용 예

- 데이터 개수 고정
- 빠른 조회가 중요할 때

---

### ✅ List (연결 리스트)

- **비연속 메모리**
- 크기 **동적**
- 중간 삽입/삭제 빠름 → **O(1)**

❌ 단점

- 특정 위치 접근 느림 → **O(N)**

📌 사용 예

- 삽입/삭제가 잦은 경우

---

### 🧠 한 줄 요약

> **배열은 조회**, **리스트는 삽입/삭제에 유리**

---

## 2️⃣ Stack (스택)

### ✅ 정의

- **LIFO (Last In, First Out)**
- 마지막에 들어온 데이터가 먼저 나감

### ✅ 주요 연산

- push / pop → **O(1)**

### 📌 사용 사례

- 함수 호출 스택
- 뒤로 가기(undo)
- 괄호 검사

### 🧠 핵심 포인트

> “되돌아가는 흐름”이 있으면 스택

---

## 3️⃣ Queue (큐)

### ✅ 정의

- **FIFO (First In, First Out)**
- 먼저 들어온 데이터가 먼저 나감

### ✅ 주요 연산

- enqueue / dequeue → **O(1)**

### 📌 사용 사례

- 프린터 대기열
- 작업 스케줄링
- BFS

### 🧠 핵심 포인트

> “순서대로 처리”하면 큐

---

## 4️⃣ Stack vs Queue (비교)

| 구분   | Stack     | Queue     |
| ------ | --------- | --------- |
| 구조   | LIFO      | FIFO      |
| 사용처 | 되돌리기  | 순서 처리 |
| 예시   | 함수 호출 | 대기열    |

# 🎤 면접 말하기 연습 (암기 스크립트)

## 🎙️ Q1. 배열과 리스트의 차이는?

> 배열은 연속된 메모리 공간을 사용해서 인덱스 접근이 O(1)로 빠르지만, 크기가 고정되어 있고 중간 삽입이나 삭제가 O(N)으로 느립니다.
> 반면 리스트는 메모리가 비연속적이라 특정 위치 접근은 O(N)이지만, 중간 삽입과 삭제가 O(1)이라 해당 작업이 많은 경우에 유리합니다.

---

## 🎙️ Q2. 스택과 큐의 차이는?

> 스택은 LIFO 구조로 마지막에 들어온 데이터가 먼저 나가는 구조이고, 큐는 FIFO 구조로 먼저 들어온 데이터가 먼저 처리됩니다.
> 스택은 함수 호출이나 되돌리기 기능에 쓰이고, 큐는 작업 대기열이나 순차 처리에 사용됩니다.

---

## 🎙️ Q3. 스택을 언제 사용하나요?

> 되돌아가야 하는 흐름이 있을 때 사용합니다. 대표적으로 함수 호출 스택, 브라우저 뒤로 가기, 괄호 검사 문제에서 사용됩니다.

---

## 🎙️ Q4. 큐를 언제 사용하나요?

> 요청을 들어온 순서대로 처리해야 할 때 사용합니다. 프린터 대기열이나 BFS 같은 탐색 알고리즘에서 큐를 사용합니다.

---

## 🧠 초단기 암기 키워드

- 배열 = **조회**
- 리스트 = **삽입/삭제**
- 스택 = **되돌리기**
- 큐 = **줄 서기**

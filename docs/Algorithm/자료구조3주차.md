> 목표: **정렬 알고리즘을 비교 설명하고, 이분 탐색 사용 조건을 명확히 말할 수 있다**

---

## 🎯 학습 목표

- 주요 정렬 알고리즘의 특징과 시간복잡도를 비교 설명할 수 있다

- 이분 탐색을 사용할 수 있는 조건을 정확히 이해한다

---

## 📚 이론 학습

### 1️⃣ Bubble Sort (버블 정렬)

#### ✅ 개념

- 인접한 두 값을 비교하며 큰 값을 뒤로 보내는 방식

#### ⏱ 시간복잡도

- 평균 / 최악: **O(N²)**

#### 🔍 특징

- 구현이 매우 간단
- **안정 정렬**

#### 🧠 한 줄 요약

> 계속 비교하며 큰 값이 거품처럼 뒤로 이동

---

### 2️⃣ Selection Sort (선택 정렬)

#### ✅ 개념

- 가장 작은 값을 선택해 앞부터 채우는 방식

#### ⏱ 시간복잡도

- 평균 / 최악: **O(N²)**

#### 🔍 특징

- 교환 횟수 적음
- **불안정 정렬**

#### 🧠 한 줄 요약

> 최소값을 찾아 위치를 확정

---

### 3️⃣ Insertion Sort (삽입 정렬)

#### ✅ 개념

- 정렬된 부분에 새 값을 알맞은 위치에 삽입

#### ⏱ 시간복잡도

- 평균 / 최악: **O(N²)**
- 거의 정렬된 경우: **O(N)**

#### 🔍 특징

- **안정 정렬**

#### 🧠 한 줄 요약

> 카드 정렬 방식

---

### 4️⃣ Merge Sort (병합 정렬)

#### ✅ 개념

- 배열을 분할 → 정렬 → 병합

#### ⏱ 시간복잡도

- 항상 **O(N log N)**

#### 🔍 특징

- **안정 정렬**
- 추가 메모리 필요

#### 🧠 한 줄 요약

> 나누고 합치며 정렬

---

### 5️⃣ Quick Sort (퀵 정렬)

#### ✅ 개념

- 기준값(Pivot)을 중심으로 분할

#### ⏱ 시간복잡도

- 평균: **O(N log N)**
- 최악: **O(N²)**

#### 🔍 특징

- 제자리 정렬
- **불안정 정렬**

#### 🧠 한 줄 요약

> 기준을 잡아 빠르게 분할

---

### 6️⃣ Heap Sort (힙 정렬)

#### ✅ 개념

- 힙 구조를 이용한 정렬

#### ⏱ 시간복잡도

- 항상 **O(N log N)**

#### 🔍 특징

- 추가 메모리 적음
- **불안정 정렬**

#### 🧠 한 줄 요약

> 우선순위를 이용한 정렬

---

## 🧠 안정 정렬 vs 불안정 정렬

### ✅ 안정 정렬

- 같은 값의 **기존 순서 유지**
- 예: Bubble, Insertion, Merge

### ❌ 불안정 정렬

- 같은 값의 순서가 바뀔 수 있음
- 예: Selection, Quick, Heap

---

## 🔍 Binary Search (이분 탐색)

### ✅ 개념

- 탐색 범위를 절반씩 줄여가는 탐색 방법

### 📌 사용 조건

- **정렬된 데이터**
- 인덱스 접근 가능

### ⏱ 시간복잡도

- **O(log N)**

#### 🧠 한 줄 요약

> 정렬된 상태에서만 가능한 빠른 탐색

---

## 🎤 기술면접 핵심 질문 & 답변

### 🎙️ Q1. 정렬 알고리즘 시간복잡도를 비교해보세요

> 버블, 선택, 삽입 정렬은 평균적으로 O(N²)이고, 병합, 퀵, 힙 정렬은 평균적으로 O(N log N)입니다.
> 특히 병합 정렬은 항상 O(N log N)을 보장합니다.

---

### 🎙️ Q2. 안정 정렬이란 무엇인가요?

> 동일한 값의 데이터가 있을 때, 정렬 전의 상대적인 순서를 정렬 후에도 유지하는 정렬을 안정 정렬이라고 합니다.

---

### 🎙️ Q3. 이분 탐색은 언제 사용 가능한가요?

> 이분 탐색은 데이터가 반드시 정렬되어 있어야 하며, 중간 값을 기준으로 탐색 범위를 줄일 수 있을 때 사용 가능합니다.

## 🧠 초단기 암기 키워드

- O(N²) = **버블 / 선택 / 삽입**
- O(N log N) = **병합 / 퀵 / 힙**
- 안정 정렬 = **버블 / 삽입 / 병합**
- 이분 탐색 = **정렬 필수**

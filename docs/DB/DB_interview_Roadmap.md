# 🗓️ 1주차 – DB 기본 개념 & RDBMS 핵심

## 🎯 1주차 목표

- 데이터베이스가 **왜 필요한지** 설명할 수 있다
- RDBMS 구조를 **그림 없이 말로 설명**할 수 있다
- 면접에서 가장 자주 나오는 **기초 질문**에 막힘없이 답변

---

## 1️⃣ 데이터베이스(Database)란?

### ✅ 핵심 개념

- **정의**: 구조화된 데이터의 집합
- **목적**: 데이터의 **영속성, 무결성, 동시성, 보안** 보장

### ❓ 면접 질문

> Q. 파일 시스템 대신 DB를 쓰는 이유는?

**모범 답변**

- 데이터 중복 감소
- 동시 접근 제어
- 무결성 제약 조건 제공
- 장애 발생 시 복구 가능

---

## 2️⃣ DBMS(Database Management System)

### ✅ 역할

- 데이터 저장 / 조회 / 수정 / 삭제(CRUD)
- 트랜잭션 관리
- 사용자 권한 관리
- 백업 & 복구

### 🔹 대표적인 RDBMS

- MySQL / PostgreSQL / Oracle / MariaDB

---

## 3️⃣ RDBMS(Relational DB) 기본 구조

### ✅ 핵심 용어 정리

| 용어         | 설명                         |
| ------------ | ---------------------------- |
| Table        | 행과 열로 구성된 데이터 집합 |
| Row (Record) | 하나의 데이터 단위           |
| Column       | 데이터 속성                  |
| Schema       | DB 구조 정의                 |
| Primary Key  | 행을 유일하게 식별           |
| Foreign Key  | 다른 테이블 참조             |

### ❓ 면접 질문

> Q. Primary Key와 Unique Key 차이점은?

**포인트**

- PK: NULL 불가 + 테이블당 1개
- Unique: NULL 허용(1개) + 여러 개 가능

---

## 4️⃣ SQL vs NoSQL (개념 수준)

### ✅ SQL (RDBMS)

- 정형 데이터
- 스키마 고정
- 트랜잭션 중요

### ✅ NoSQL

- 비정형/반정형 데이터
- 스키마 유연
- 확장성 중시

> ⚠️ 1주차에서는 **차이만 이해**, 깊게 파지 않음

---

## 5️⃣ 무결성(Integrity)

### ✅ 무결성 종류

- **개체 무결성**: PK는 NULL 불가
- **참조 무결성**: FK는 참조 대상 존재
- **도메인 무결성**: 컬럼 타입/범위 제한

### ❓ 면접 질문

> Q. 무결성을 왜 보장해야 하나요?

→ 데이터 신뢰성 확보
